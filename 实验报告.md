## 任务说明

本次大作业基于 Amazon Reviews 2023 数据集完成 Top-K 商品推荐任务。按照作业说明，实验使用三个 5-core 类别数据集：

- Industrial_and_Scientific
- Musical_Instruments
- CDs_and_Vinyl

用户交互序列按时间顺序划分为 train、dev、test，模型根据用户历史交互预测测试集中下一个可能交互的商品。最终评价指标采用测试集 NDCG@10，同时记录 Recall@10、Recall@50 和 NDCG@50 作为辅助指标。

## 基线方法

原始代码中主要包含两个模型：

- SASRecText：在 SASRec 序列推荐框架上使用商品文本向量。
- UniSRec：使用预训练文本编码得到的商品表示，并通过序列模型进行推荐。

根据原始日志，UniSRec 在三个数据集上的测试集 NDCG@10 均优于 SASRecText，因此本实验以 UniSRec 作为主要对比基线。

| 数据集 | 原始 UniSRec Test NDCG@10 | 原始 UniSRec Test Recall@10 |
| --- | ---: | ---: |
| Musical_Instruments | 0.0340 | 0.0650 |
| Industrial_and_Scientific | 0.0267 | 0.0524 |
| CDs_and_Vinyl | 0.0368 | 0.0740 |

## 优化思路

本实验的优化主要从三方面展开。

第一，补充纯序列推荐模型。除原始文本增强模型外，额外加入 SASRec、GRU4Rec、NARM 等 ID-only 序列模型。不同模型对用户短期兴趣、长期兴趣和序列转移关系的建模方式不同，能够提供互补排序结果。

第二，扩展商品侧元数据。原始模型主要使用商品文本信息，本实验进一步合并 title、main category、store、categories、features、description、details、price、average rating、rating number 和 rank 等字段，构造更丰富的商品文本与结构化特征。

第三，加入用户历史侧特征。除 item_id 序列外，额外引入历史评分、时间间隔和 recency 等序列特征，使模型能够利用用户历史行为的强弱和时间变化。

在上述基础上，新增 RichUniSRec 模型。该模型融合 rich item text embedding、可训练 item ID embedding、结构化商品特征 embedding，以及用户历史评分和时间特征。最后，对多个模型的 Top-K 排序结果使用 RRF（Reciprocal Rank Fusion）进行融合，降低单模型波动带来的影响。

## 模型与实现

代码基于 RecBole 框架实现，主要新增或修改内容如下：

- `run.py`：支持更多模型名称、实验参数覆盖、结果 JSONL 记录和 checkpoint 备份。
- `config/SASRec.yaml`、`config/GRU4Rec.yaml`、`config/NARM.yaml`：新增 ID-only 模型配置。
- `dataset/process_amazon.py`：支持 rich metadata 处理，并生成历史评分、时间间隔和 recency 序列。
- `data/dataset.py`：新增 rich dataset 读取逻辑。
- `model/richunisrec.py`：实现 RichUniSRec。
- `scripts/export_ranklist.py`：导出测试集 Top-K 排序结果。
- `scripts/rrf_fusion.py`：实现 RRF 排序融合。

RRF 融合使用各模型的排序名次进行加权，不直接依赖不同模型输出分数的尺度，因此适合融合 UniSRec、RichUniSRec 和 ID-only 模型。

## 实验设置

主要实验环境如下：

- Python 3.10
- PyTorch 2.1.x
- CUDA 12.x
- RecBole 1.2.1
- GPU：NVIDIA RTX 4090 / 4090D

实验过程中保留 train、dev、test 三份划分。训练阶段使用 dev 集进行 early stopping，最终在 test 集上报告 NDCG@10。所有实验日志、结果文件和模型 checkpoint 均已保存，便于复现。

## 最终结果

PDF 指定的主要指标为测试集 NDCG@10。最终结果如下：

| 数据集 | 最终方法 | Test Recall@10 | Test NDCG@10 | Test Recall@50 | Test NDCG@50 |
| --- | --- | ---: | ---: | ---: | ---: |
| Musical_Instruments | RRF(UniSRec + SASRec + GRU4Rec + NARM + RichUniSRec) | 0.0739 | 0.0383 | 0.1788 | 0.0610 |
| Industrial_and_Scientific | RRF(UniSRec + SASRec + GRU4Rec + NARM + RichUniSRec) | 0.0612 | 0.0311 | 0.1469 | 0.0497 |
| CDs_and_Vinyl | RRF(RichUniSRec + SASRec + GRU4Rec + NARM) | 0.1166 | 0.0553 | 0.2407 | 0.0825 |

与原始 UniSRec 基线相比，提升如下：

| 数据集 | 原始 UniSRec NDCG@10 | 最终 NDCG@10 | 绝对提升 | 相对提升 |
| --- | ---: | ---: | ---: | ---: |
| Musical_Instruments | 0.0340 | 0.0383 | +0.0043 | +12.6% |
| Industrial_and_Scientific | 0.0267 | 0.0311 | +0.0044 | +16.6% |
| CDs_and_Vinyl | 0.0368 | 0.0553 | +0.0185 | +50.1% |
| 平均 | 0.0325 | 0.0416 | +0.0091 | +27.9% |

## 结果分析

从结果看，三个数据集的 NDCG@10 均有提升。其中 CDs_and_Vinyl 提升最明显，说明在商品数量较多、交互序列较复杂的数据集上，补充 ID-only 序列模型和 richer metadata 后，模型能够获得更强的排序能力。

Musical_Instruments 和 Industrial_and_Scientific 的提升相对较小，但仍稳定超过原始 UniSRec。主要原因是原始 UniSRec 已经使用了商品文本信息，单独调整文本模型收益有限；加入用户侧历史评分、时间特征以及多模型融合后，能够进一步改善 Top-K 排序。

RRF 融合的优势在于不同模型错误不完全一致。文本增强模型更依赖商品语义，ID-only 模型更依赖交互共现和序列转移，融合后能提升推荐列表前 10 位的命中质量。

## 总结

本实验在原始 UniSRec 基线基础上，加入了更多序列模型、扩展商品侧 metadata、补充用户历史评分与时间特征，并使用 RRF 对多个模型排序结果进行融合。最终三个数据集的测试集 NDCG@10 均超过原始 UniSRec，平均 NDCG@10 从 0.0325 提升到 0.0416，相对提升约 27.9%。
